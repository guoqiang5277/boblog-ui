    图片 (Image)

    图片展示

    • 基础类名: .boblog-image
    • 边框: 1px solid --boblog-border
    • 图片样式: .boblog-image img (block，max-width 100%，高度自适应)
    • 说明文字: .boblog-image-caption (居中灰色小字，顶部分割线)
    • 尺寸变体: .boblog-image-sm (150px) / .boblog-image-md (300px) / .boblog-image-lg (500px) / .boblog-image-full (100%)
    • 样式变体: .boblog-image-bordered (蓝色边框) / .boblog-image-shadow (右下角阴影)
    • 浮动布局: .boblog-image-float-left (左浮文字环绕) / .boblog-image-float-right (右浮文字环绕)

    基础图片（带说明文字）：

    300×200
    这是一张示例图片的说明文字

    尺寸变体（sm / md / lg / full）：

    Small 150px
    Medium 300px
    Large 500px
    Full Width 100%

    样式变体（边框 / 阴影）：

    蓝色边框
    带阴影

    浮动布局（左浮 / 右浮，文字环绕）：

    左浮动

    这是环绕在图片右侧的文字内容。左浮动的图片会让文字自动环绕在其右侧和下方， 适合图文混排的博客文章场景。这种布局方式在经典的 Bo-Blog 文章编辑器中非常常见， 可以让图片与文字更紧密地结合在一起，提升阅读体验。文字会自然地流动到图片周围， 形成美观的排版效果。

    右浮动

    这是环绕在图片左侧的文字内容。右浮动的图片会让文字自动环绕在其左侧和下方， 与左浮动相反。这种排版方式同样适合博客文章，可以根据版面需要灵活调整图片位置。 文字会自然地填充图片左侧的空间，保持紧凑的视觉效果。

    画廊相册

    • 默认列宽: 最小 180px，自动填充可用宽度

    • 单个画廊项: .boblog-gallery-item
    • 画廊项图片: .boblog-gallery-item img (宽度 100%，固定高度 180px，object-fit cover)
    • 说明文字: .boblog-gallery-item-caption

    幻灯片

    • 容器类名: .boblog-slideshow
    • 实现方式: CSS scroll-snap + JS 交互控制
    • 滑动轨道: .boblog-slideshow-track (flex 横向排列，隐藏滚动条)
    • 单张幻灯片: .boblog-slideshow-slide (宽度 100%，scroll-snap-align start)
    • 说明文字: .boblog-slideshow-caption (底部半透明黑色遮罩，白色文字)
    • 指示点: .boblog-slideshow-dots > .boblog-slideshow-dot (小圆点，JS 自动更新 active)
    • 切换箭头: .boblog-slideshow-prev / .boblog-slideshow-next (两侧半透明箭头，循环切换)
    • 尺寸变体: .boblog-slideshow-sm (400px) / .boblog-slideshow-lg (800px) / .boblog-slideshow-full (100%)
    • 交互方式: 鼠标拖拽/触屏滑动、点击箭头循环切换、点击指示点跳转，滚动后自动更新焦点

    幻灯片 JavaScript API

    1. 自动初始化

    页面加载后自动初始化所有 .boblog-slideshow 组件。JS 负责：指示点点击跳转并更新 active 状态、箭头循环切换（末尾→第一张）、滚动/滑动后自动检测当前幻灯片并同步指示点。

    2. 手动调用

    // 初始化指定容器内的幻灯片（用于动态插入的内容）
BoblogUI.image.init(document.getElementById('myContainer'));

    3. 引入方式

    <link rel="stylesheet" href="../boblog-ui.css">
<script src="../boblog-ui.js"></script>
