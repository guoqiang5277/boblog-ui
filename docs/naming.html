    命名规范

    类名前缀

    所有类名统一使用 boblog- 前缀：

    • boblog-btn — 按钮
    • boblog-input — 输入框
    • boblog-table — 表格
    • boblog-editor — 编辑器

    组件 vs 控件

    类型 判断标准 目录 示例
    组件 (components) 需要 JS 初始化或有交互行为 src/components/ editor, tabs, toc, datepicker, codeblock
    控件 (controls) 纯 CSS 实现 src/controls/ button, input, select, table, card, badge

    判断依据：有对应的 src/js/*.js 文件 → 组件；仅有 CSS 文件 → 控件。

    类名结构

    结构 用途 示例
    boblog-{名称} 组件/控件根类 boblog-btn、boblog-table
    boblog-{名称}-{变体} 变体/修饰类 boblog-btn-primary、boblog-table-bordered
    boblog-{名称}-{子元素} 子元素类 boblog-editor-content、boblog-modal-header

# boblog-ui 目录结构规范

## src 目录结构

```
src/
├── base/           # 基础
├── controls/       # 基础控件
├── components/     # 复合组件
├── layout/         # 布局
├── frontend/       # 前台专用
└── js/             # JavaScript
```
