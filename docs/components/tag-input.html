    标签输入器 (Tag Input)

    基础用法

    • 最外层容器: .boblog-tag-input（相对定位，作为下拉面板的锚点）
    • 输入区域: .boblog-tag-input-wrapper .boblog-input（flex 布局，包含已选标签和输入框）
    • 已选标签列表: .boblog-tag-input-tags（flex 布局，自动换行）
    • 单个标签: .boblog-badge.primary（复用 badge 组件，蓝底白字）
    • 删除按钮: .boblog-tag-input-remove（标签内的 × 按钮）
    • 文本输入框: .boblog-tag-input-field（无边框透明背景，融入 wrapper）
    • 下拉建议面板: .boblog-tag-input-suggestions（绝对定位，默认 display:none）
    • 单条建议项: .boblog-tag-input-option（flex 布局，名称和计数左右分布）
    • 标签名: .boblog-tag-input-name（占据剩余空间）
    • 使用次数: .boblog-tag-input-count（灰色小字，右侧对齐）
    • 新建标签提示: .boblog-tag-input-new（绿色小字，仅在无完全匹配时显示）

    在输入框中键入内容可触发建议列表；按 Enter 或逗号确认添加标签；点击标签内的 × 删除；Backspace 删除最后一个标签。

    API 说明

    • 全局对象: BoblogTagInput（挂载在 window 上，由 boblog-ui.js 提供）
    • 入口方法: BoblogTagInput.create(config) — 初始化并返回实例
    • 返回值: 实例对象，包含 addTag / removeTag / getSelectedTags 公共方法

    create() 配置参数

    参数 类型 必填 说明
    container HTMLElement 是 最外层 .boblog-tag-input 容器 DOM 元素
    availableTags Array 否 可用标签列表，每项格式：{ id, name, categoryName?, articleCount? }。输入时在此列表中搜索建议。
    selectedTags Array 否 初始已选标签列表，每项格式：{ id?, name }。默认为空数组。
    onTagsChange Function 否 标签变化回调，签名：function(selectedTags)。每次添加或删除标签后触发，参数为当前已选标签数组的副本。
    renderOption Function 否 自定义渲染建议项，签名：function(tag, inputValue) → HTML string。返回完整的 .boblog-tag-input-option HTML 字符串。未传则使用内置渲染逻辑。

    availableTags 字段说明

    字段 类型 必填 说明
    id number / string 否 标签 ID，选中后会携带到 selectedTags 中
    name string 是 标签名称，用于搜索匹配和显示
    categoryName string 否 所属分类名，显示在建议项左侧，以小型 badge 形式呈现
    articleCount number 否 使用该标签的文章数，显示在建议项右侧（格式：N 篇文章）

    实例公共方法

    方法 参数 说明
    addTag(name, id) name: string, id?: number 程序化添加标签。已存在同名标签时自动去重，添加后触发 onTagsChange。
    removeTag(name) name: string 程序化删除指定名称的标签，删除后触发 onTagsChange。
    getSelectedTags() 无 返回当前已选标签数组的副本（不影响内部状态）。

    键盘交互

    • Enter：若有高亮建议项则选中该项；否则将输入框文字作为新标签添加
    • 逗号（, 或 ，）：将逗号前的文字作为新标签添加
    • Backspace：输入框为空时删除最后一个已选标签
    • ArrowDown / ArrowUp：在建议列表中上下导航，循环选中
    • Escape：关闭建议面板
    • 点击外部区域：关闭建议面板

    新建标签逻辑

    • 输入内容与 availableTags 无完全匹配时，建议面板末尾自动追加"+ 创建新标签: ..."选项
    • 新建标签的 id 为 null，name 为当前输入值
    • 通过 onTagsChange 回调可识别 id 为 null 的项，在保存时向后端创建新标签
