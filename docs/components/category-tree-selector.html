    分类树形选择器 (Category Tree Selector)

    概述

    后台管理系统中用于选择文章分类的交互组件。提供下拉树形选择器，替代传统的下拉列表。

    • 树形展示：以层级结构显示所有分类，直观展示父子关系，最多支持4级嵌套
    • 路径显示：选中分类后显示完整的分类路径（如：技术 > 前端 > Vue）
    • 搜索过滤：支持按名称、别名、描述搜索，支持拼音搜索
    • 折叠展开：可折叠/展开有子分类的节点
    • 快速新建：可直接在选择器中新建分类（内置弹窗）
    • 单选模式：选中后自动关闭下拉面板

    依赖：

    • BoblogTreeSelect（tree-select.js）通用树形选择器
    • BoblogUI.pinyin 拼音搜索模块

    类名前缀

    • 主选择器使用 boblog-tree-select- 通用前缀（复用 BoblogTreeSelect 库）
    • 内部结构元素使用无前缀的 tree-item-* 类名
    • 父分类选择器（新建弹窗内）同样使用 boblog-tree-select- 前缀

    搜索规则

    搜索框支持三种匹配方式，依赖 BoblogUI.pinyin 模块（已内置于 boblog-ui.js）：

    匹配方式示例说明
    直接文本输入"前端" → 匹配"前端"搜索词直接包含在分类名称或别名中
    拼音首字母输入"qd" → 匹配"前端"搜索词为目标文本的拼音首字母序列
    完整拼音输入"qianduan" → 匹配"前端"搜索词为目标文本的完整拼音拼写

    匹配范围：依次匹配分类名称（data-name）和别名（.boblog-tree-select-slug），任一命中即显示。

    详细 API 参见 拼音搜索模块文档。

    两层架构

    层级模块职责
    库层 BoblogTreeSelect
    boblog-ui.js 内置    		 通用树形选择器引擎：树形渲染、展开折叠、搜索过滤（拼音）、节点选中、面板开关、键盘操作
    业务层 CategoryTreeSelector
    category-tree-selector.js    		 分类专有业务逻辑：调用 BoblogTreeSelect.create() 创建主选择器和父分类选择器、新建分类弹窗（AJAX）、隐藏字段同步、类型徽章

    调用方式

    调用方只需要接触业务层，不需要直接调用 BoblogTreeSelect.create()：

    // 调用方只需一行
CategoryTreeSelector.init({
    categoryTree: treeData,
    selectedId: null
});

// init() 内部自动完成：
// 1. BoblogTreeSelect.create() 创建主选择器（含类型徽章、路径显示、搜索配置）
// 2. BoblogTreeSelect.create() 创建父分类选择器（弹窗内）
// 3. 绑定新建弹窗事件
// 4. 同步隐藏字段 #CategoryId

    1. 引入文件

    <!-- boblog-ui 库（CSS 样式 + BoblogTreeSelect JS + 拼音搜索） -->
<link rel="stylesheet" href="boblog-ui.css" />
<script src="boblog-ui.js"></script>

<!-- 分类选择器业务脚本 -->
<script src="category-tree-selector.js"></script>

    2. 放置 HTML

    选择器和新建弹窗是两个独立的 DOM 结构，详细 HTML 见"HTML 结构"Tab。

    3. 初始化

    // 分类树数据（通常由后端 API 返回）
var categoryTree = [
    { id: 1, name: '技术', slug: 'tech', type: 'directory', children: [
        { id: 2, name: '前端', slug: 'frontend', type: 'final', description: 'HTML/CSS/JavaScript', articleCount: 12, children: [] },
        { id: 3, name: '后端', slug: 'backend', type: 'final', description: 'C#/.NET/数据库', articleCount: 8, children: [] }
    ]},
    { id: 4, name: '生活', slug: 'life', type: 'final', articleCount: 5, children: [] },
    { id: 5, name: '随笔', slug: 'essay', type: 'final', articleCount: 3, children: [] }
];

// 初始化
CategoryTreeSelector.init({
    categoryTree: categoryTree,
    selectedId: null              // 编辑页传文章已有的分类 ID，新建页传 null
});

    必须的 DOM 元素

    BoblogTreeSelect.create() 通过以下选择器查找 DOM，这些元素必须存在：

    选择器说明
    .boblog-tree-select-display显示框（点击打开/关闭下拉面板）
    .boblog-tree-select-dropdown下拉面板容器
    .boblog-tree-select-search input搜索输入框
    .boblog-tree-select-list树节点渲染容器（JS 在此生成节点 HTML）
    [data-display-name]显示选中名称的元素

    可选的 DOM 元素

    选择器说明
    [data-display-path]显示选中路径（配合 showPath: true 使用）
    #btn-create-category新建按钮（业务层绑定事件，库层不关心）
    #CategoryId隐藏字段（业务层同步选中 ID，库层不关心）

    BoblogTreeSelect.create(config) 配置项

    库层 API，由业务层 CategoryTreeSelector.init() 内部调用，调用方通常不需要直接使用。

    配置项类型默认值说明
    containerHTMLElement—必填。最外层 .boblog-tree-select 容器元素
    treeDataArray[]必填。树形数据（见"数据格式"Tab）
    selectedIdnumber | nullnull初始选中的节点 ID
    placeholderstring'请选择'未选中时显示框的占位文字
    emptyOptionTextstring | nullnull空选项文字（如"不选择分类"），null 则不显示空选项
    showPathbooleanfalse选中后是否在显示框下方显示完整路径
    onSelectFunction空函数选中回调 function(id, name, fullPath)
    isSelectableFunction叶子节点可选判断节点是否可选 function(node) → boolean。不可选的节点置灰显示
    nodeMatchesFunction | nullnull（使用默认拼音匹配）自定义搜索匹配 function(node, search) → boolean
    renderNodeExtraFunction | nullnull自定义节点额外 HTML function(node) → string，如类型徽章

    实例方法

    BoblogTreeSelect.create() 返回的实例对象上可调用：

    方法说明
    getSelectedId()获取当前选中的节点 ID
    selectById(id)程序化选中指定节点
    refreshData(treeData)用新的树数据刷新选择器（新建分类后使用）

    通过 CategoryTreeSelector.mainSelector 可访问主选择器实例，CategoryTreeSelector.parentSelector 可访问父分类选择器实例。

    treeData 数据结构

    传入 categoryTree 的数据格式，后端 API 需返回此结构的 JSON：

    [
    {
        id: 1,
        name: "技术",
        slug: "tech",
        type: "directory",       // "directory"（目录型）或 "final"（终极型）
        description: "",
        articleCount: 0,
        children: [
            {
                id: 2,
                name: "前端",
                slug: "frontend",
                type: "final",
                description: "HTML/CSS/JavaScript",
                articleCount: 12,
                children: []
            },
            {
                id: 3,
                name: "后端",
                slug: "backend",
                type: "final",
                description: "C#/.NET/数据库",
                articleCount: 8,
                children: []
            }
        ]
    },
    {
        id: 4,
        name: "生活",
        slug: "life",
        type: "final",
        articleCount: 5,
        children: []
    }
]

    节点字段说明

    字段类型必填说明
    idnumber是分类 ID
    namestring是分类名称
    slugstring否别名（用于 URL），显示在节点名称右侧
    typestring否"directory" 或 "final"（默认 "final"）。目录型不可选，终极型可选
    descriptionstring否分类描述，显示在节点名称下方，搜索时也会匹配
    articleCountnumber否文章数量，显示为计数徽章
    childrenarray是子分类数组，无子分类时为空数组 []

    数据输入（初始化参数）

    CategoryTreeSelector.init(config) 接受的参数：

    参数类型说明
    categoryTreeArray分类树形数据（见"数据格式"Tab），通常由后端接口返回
    selectedIdnumber | null当前选中的分类 ID。编辑页传文章已有的分类 ID，新建页传 null

    数据输出（用户操作后）

    用户选中分类后，业务层自动将选中的分类 ID 写入隐藏字段：

    <input type="hidden" name="CategoryId" id="CategoryId" value="2" />

    表单提交时，CategoryId 随表单一起提交到后端。选择"不选择分类"时 value 为空字符串。

    新建分类（AJAX）

    用户点击"+"按钮打开新建弹窗，填写后提交：

    步骤说明
    请求POST 到后端创建接口，提交 name、slug、parentId、description、type
    成功后新节点插入 treeData，调用 refreshData() 刷新两个选择器，终极型自动选中
    失败时显示后端返回的错误信息到对应字段
