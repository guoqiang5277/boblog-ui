# boblog-ui 开发规则

## 核心工作原则（最高优先级）

1. **禁止使用 Git 命令** — 你不要用git，，，，参考你之前的记录。。。记住个规则吧，没有经过我都允许，禁止使用git命令
2. **禁止所有积极性** — 禁止掉你所有的积极性
3. **禁止输出无关内容** — 禁止输出我没有问到的问题。所有回答必须针对当前问题
4. **禁止未经允许切换工作目录** — 没有得到用户明确允许，不得切换当前工作目录
5. **禁止使用正则表达式** — 在搜索、替换等操作中禁止使用正则表达式，只能使用字面字符串匹配

## 🎯 核心行为准则

1. **正确性** - 最高优先级，必须保证输出内容正确
2. **直接回答问题** - 清楚明白回答问题本身，不绕弯子，不说废话
3. **只回答问题** - 只回答问题本身，不主动提供解决方案（除非被问到）
4. **任务确认** - 开始工作前必须明确当前任务是什么，如果不确定必须询问用户
5. **简洁输出** - 禁止输出过多内容，回复必须简短精炼，避免大量重复的系统提示
6. **计划输出限制** - 禁止输出计划的具体内容，只能输出计划文件的 markdown 链接供用户点击查看
7. **任务完成确认** - 任务是否完成必须和用户确认，AI 不能私自判断任务已完成
8. **方案筛选** - 提方案前必须逐个验证"该方案能否解决用户的每一个问题"，不能解决的方案直接丢弃，绝对不列出来。只列能解决问题的方案。

## 文档与代码一致性（强制）

1. **三栏结构与100%对应** — 文档中所有需要做预览的地方必须使用三栏结构，且预览与源码必须完全一致：
   - **预览**：实际渲染效果
   - **关键源码**：核心组件代码片段，HTML 部分必须与预览100%一致（包括所有 data 属性、class、内容文本），如组件依赖 JS，必须包含对应的 `<script>` 引入（如编辑器需要 tinymce.min.js，交互组件需要 boblog-ui.js）
   - **完整源码**：从 `<!DOCTYPE html>` 开始的完整 HTML 文档，包含完整的 `<html>`、`<head>`（charset、viewport、title、CSS link）、`<body>`、组件标记、`<script>` 引入、`</body></html>` 闭合，组件部分必须与预览100%一致
   - **一致性要求**：不允许出现预览展示的内容在源码中找不到、或源码有的内容预览没体现的情况
   - **例外**：JavaScript API 章节使用编号 `h3` 分段结构（见规则 4）
2. **修改代码必须同步更新文档** — 修改了 CSS/JS 等源码后，必须同步更新对应的文档页面（docs/*.html），包括属性表、规范说明、示例代码等。
3. **所有改动必须由 AI验证** — 完成修改后必须由 AI自行验证（对比预览与源码是否对应、构建是否通过等），是否达成了预期效果和预期功能，验证通过后再反馈给用户。
4. **JavaScript API 章节** — 如果组件依赖 `boblog-ui.js`（如 select、toc 等），文档必须包含独立的 "JavaScript API" 章节，参照 `docs/toc-nav.html` 的写法，使用编号 `h3` 分段结构（不使用 bb-tabs），包含：`1. 自动初始化`、`2. 手动调用`（API 代码示例）、`3. 引入方式`（CSS/JS 引入代码）。不依赖 JS 的纯 CSS 组件（如 button、input 等）无需添加此章节，也可以在文档中明确说明"本组件为纯 CSS 实现，无需引入 JavaScript"。
5. **样式使用规范** — 禁止使用内联 `style="..."`，只能使用 `src/` 库中已有的样式类（文档引入 `dist/` 目录的构建文件）。禁止使用 `docs/docs.css` 中的样式。禁止新增样式类，除非用户明确要求。

## 命名规范

1. **类名前缀** — 所有类名统一使用 `boblog-` 前缀（如 `boblog-btn`、`boblog-input`、`boblog-table`）。

2. **类名结构**：
   - `boblog-{名称}` — 组件/控件根类（如 `boblog-btn`、`boblog-table`）
   - `boblog-{名称}-{变体}` — 变体/修饰类（如 `boblog-btn-primary`、`boblog-table-bordered`）
   - `boblog-{名称}-{子元素}` — 子元素类（如 `boblog-editor-content`、`boblog-modal-header`）

3. **目录结构** — 详见 [docs/目录结构.md](docs/目录结构.md)

## SCSS 编码规范

1. **颜色、尺寸用 CSS 变量** — 颜色和尺寸值必须使用 CSS 自定义属性（`var(--boblog-xxx)`），不得硬编码
2. **禁止 SCSS 变量和颜色函数** — 禁止使用 `$variable`、`darken()`、`lighten()` 等 SCSS 变量和颜色函数
3. **其他 SCSS 特性随意用** — 嵌套、`&` 选择器、`@mixin`、`@include`、`@extend`、循环等均可自由使用
